# 会員制 + AI 質問チャット 設計計画

> ステータス: **設計フェーズ**(まだコードは存在しない)
> 作成日: 2026-04-29

機能 3「AI 質問チャット」を有料会員機能として導入するための実装計画。**着手前にいくつか決定が必要**。

---

## 1. ゴール

- 無料: これまで通りすべて閲覧 + 練習可能(現状維持)
- 有料: **教科書本文をコンテキストにした AI 質問**ができる
- 月額数百円 〜 数千円のサブスクで運営費を回収

---

## 2. 全体アーキテクチャ案

```
[ユーザー]
   ↓
[Next.js Page]
   ↓ 認証チェック
[Clerk]                  ← Vercel Marketplace から install
   ↓ 認証 OK
[Server Action / API]
   ↓
[Stripe API]             ← 会員ステータス確認
   ↓ 有料会員のみ
[Vercel AI Gateway]      ← Claude / GPT 切り替え可能
   ↓
[ユーザーに回答ストリーム]
```

会話履歴の保存をするなら別途 DB が必要(Vercel Postgres or Neon)。

---

## 3. 必要な決定事項(ユーザー側)

### 3-1. 認証プロバイダ

| 候補 | 月額 | 特徴 |
|---|---|---|
| **Clerk** | 月 1,000 アクティブまで無料 | UI コンポーネント完備・最速 |
| Auth0 | 月 7,500 まで無料 | エンタープライズ機能 |
| Supabase Auth | 月 50,000 まで無料 | DB と一体型・コスト最安 |

**推奨**: Clerk(Vercel Marketplace から 1 クリック設定可能) → 決定後に着手

### 3-2. 決済プロバイダ

| 候補 | 手数料 | 特徴 |
|---|---|---|
| **Stripe** | 3.6% | 業界標準・サブスク管理が充実 |
| Paddle | 5% | 税金処理を代行(海外向けに楽) |
| Clerk Billing | Stripe ラッパー | Clerk 認証と統合済み |

**推奨**: Clerk Billing (Stripe を抽象化したもの)

### 3-3. AI プロバイダ

**Vercel AI Gateway** を経由して以下を選択(切り替え可):
- Claude Opus 4.7 (品質最高)
- Claude Sonnet 4.6 (バランス型・推奨)
- GPT-4o (互換)
- 安価なモデル(Haiku 等)

### 3-4. データベース(チャット履歴を保存する場合)

| 候補 | 無料枠 |
|---|---|
| **Neon Postgres** | 0.5 GB 無料 |
| Vercel Postgres | (Neon ベース) |
| Supabase | 500 MB 無料 |

**推奨**: Neon (Vercel Marketplace 経由)

### 3-5. 価格設定

| プラン候補 | 月額 | 含まれるもの |
|---|---|---|
| 無料 | 0 円 | サイト全機能(教科書・問題・図解・進捗) |
| **Pro** | **480 円** | 月 200 質問・履歴 30 日保存 |
| **Ultra** | **980 円** | 月 1,000 質問・履歴無制限・優先処理 |

→ 数値は要相談。AI 利用料(Claude Opus 1質問≈3〜10円)から逆算する。

### 3-6. 利用規約 / プライバシーポリシー更新

- 個人情報取得(メアド) → プライバシーポリシー追記が必須
- 決済情報取扱(Stripe 経由) → 規約に明記
- AI 出力の責任範囲 → 規約必須
- 解約 / 返金ポリシー → 規約必須

→ 必要なら雛形を生成可能。

## 4. 実装ステップ(着手後の想定)

### Phase 1: 認証基盤(1〜2 日)
- Vercel Marketplace から Clerk install
- `<SignInButton>` `<UserButton>` 配置
- `/sign-in` `/sign-up` ページ
- Middleware でログイン要否を制御

### Phase 2: 決済(2〜3 日)
- Clerk Billing でプラン定義
- `/pricing` ページ
- Webhook で会員ステータス同期
- ユーザーオブジェクトに `plan: "free" | "pro" | "ultra"` を持たせる

### Phase 3: AI チャット(3〜5 日)
- AI SDK + Vercel AI Gateway 接続
- `/chat` ページ(Pro 以上限定)
- 教科書本文を RAG コンテキストとして埋め込み
- ストリーミング応答
- レート制限(月クオータ管理)

### Phase 4: チャット履歴(2 日)
- Neon に `messages` テーブル
- ユーザーごとの履歴一覧 UI
- 検索 / お気に入り

### Phase 5: 運用(継続)
- Vercel AI Gateway のダッシュボードで月次コスト監視
- ユーザーフィードバック収集
- 不適切利用検知(必要なら)

---

## 5. リスクと注意点

| リスク | 対策 |
|---|---|
| AI 利用料が想定超え | 月クオータ + アラート + 上限ガード |
| 不正利用(API 経由スクレイピング) | レート制限 + Vercel BotID |
| プロンプトインジェクション | システムプロンプトで防御 + 出力モニタリング |
| 退会ユーザーのデータ残存 | GDPR 対応 → 削除エンドポイント実装 |
| Stripe 手数料で赤字 | 価格設計時に逆算済みであれば問題なし |

---

## 6. ローンチ後 KPI

- 有料転換率(目標: 訪問者の 0.5〜2%)
- 月間チャット利用率(Pro 会員のうち月 1 回以上使う %)
- 解約率(月 5% 以下が目安)
- 平均応答評価(👍/👎)
